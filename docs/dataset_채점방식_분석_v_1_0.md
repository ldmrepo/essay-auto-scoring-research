## 1. 채점 구조

### 대분류 3개

| 대분류 | JSON 키 (`essay_scoreT_detail.*`) |
|---|---|
| 표현 | `essay_scoreT_exp` |
| 구성 | `essay_scoreT_org` |
| 내용 | `essay_scoreT_cont` |

### 소분류 항목 (0~3점 척도)

**표현**
- 문법의 정확성
- 단어 사용의 적절성
- 문장 표현의 **적절성** (논술형) / 문장 표현의 **탄력성** (수필형)

**구성**
- 문단 내 구조의 적절성
- 문단 간 구조의 적절성
- 구조의 일관성
- 분량의 적절성

**내용**
- 주제의 명료성
- 설명의 **구체성** (논술형) / 설명의 **풍부성** (수필형)
- 사고의 창의성
- 프롬프트 독해력 *(논술형 전용; 수필형은 자유 주제로 평가 항목 없음)*

> 명칭 차이는 학년군이 아니라 **에세이 타입**(논술형 vs 수필형)에 따라 달라집니다.

### 에세이 타입 2종

| 타입 | 종류 | 데이터셋 5종 |
|---|---|---|
| 논술형 | 주장 / 찬성반대 / 대안제시 | 3종 |
| 수필형 | 설명글 / 글짓기 | 2종 |

### 학년군 4종 (PDF spec) + 데이터셋 학년 매핑

| 학년군 (PDF) | 데이터셋 실제 학년 |
|---|---|
| 초4~초5 | 초4, 초5 |
| 초6~중1 | 초6, 중1 |
| 중2~고1 | 중2, **중3**, 고1 (PDF에 중3 미명시이나 실제 데이터에 존재) |
| 고2~고3 | 고2, **고3** (수필형/대안제시에만 존재, 논술형 고3 없음) |

데이터셋 학년 분포 (검증 시 확인):
- 중3: 찬성반대 107편, 주장 216편, 대안제시 109편, 설명글 192편, 글짓기 117편
- 고3: 설명글 97편, 글짓기 92편

→ PDF의 4개 학년군 + 데이터의 6개 학년 (초4·초5·초6·중1·중2·중3·고1·고2·고3)의 매핑은 `rubric` 가중치 패턴으로 자동 식별 가능.

## 3. 점수 계산 흐름

> ⚠️ 공식은 PDF / 데이터 설명서에 명시되지 않았으며, 실 데이터에서 역추정한 것입니다 (ESSAY_69855 rater 2로 정확 일치 검증 완료).

```
1. 채점자(3명 독립) 각 활성 소분류에 0~3점 부여
   (가중치 0인 슬롯은 평가 미적용)
        ↓
2. 같은 대분류 안에서 정규화:
   대분류_점수 = Σ(소분류점수 × 소분류가중치) / Σ(소분류가중치)
   → 0~3점 범위
        ↓
3. 대분류 점수 × 대분류 가중치 (논술형 3:3:4, 수필형 4:2:4; 합 10)
        ↓
4. 3개 대분류 가중점수 합산 → 채점자 1인 총점 (0~30점)
        ↓
5. essay_scoreT [채점자1 총점, 채점자2 총점, 채점자3 총점]
        ↓
6. 산술 평균 → essay_scoreT_avg (0~30점)
```

### 검증 예 (ESSAY_69855, 고2~고3 논술형)

채점자 2의 모든 점수 균등 (exp=[3,3,3], org=[3,3,3,3], cont=[3,3,3,3]):
- exp_macro = (3×2 + 3×4 + 3×4) / (2+4+4) = 30/10 = 3.0
- org_macro = (3×3 + 3×3 + 3×3 + 3×1) / 10 = 3.0
- cont_macro = (3×3 + 3×3 + 3×3 + 3×1) / 10 = 3.0
- total = 3.0×3 + 3.0×3 + 3.0×4 = 9 + 9 + 12 = **29.4** ✓ (JSON `essay_scoreT[1]` = 29.4 일치)

### 검증 예 (essay_scoreT_avg)

ESSAY_69855: (24.5 + 29.4 + 23.1) / 3 = 25.6667 → JSON `25.666666` 일치
글짓기_고등_3학년_ESSAY_40773: (30 + 28.5 + 30) / 3 = 29.5 → JSON `29.5` 일치

## 4. JSON 구조

### 실 데이터 예 (ESSAY_69855, 찬성반대 고2 → **고2~고3 논술형** 학년군)

```json
"essay_scoreT_detail": {
  "essay_scoreT_exp":  [[2,3,3], [2,3,3], [2,2,3]],
  // 표현 슬롯은 항상 3개 (학년군과 무관). 가중치 0인 슬롯은 무시
  "essay_scoreT_org":  [[2,2,3,2], [3,3,3,3], [2,2,2,3]],
  // 구성 슬롯은 항상 4개
  "essay_scoreT_cont": [[2,2,2,3], [3,3,3,3], [3,2,3,2]]
  // 내용 슬롯은 항상 4개
},
"essay_scoreT":     [24.5, 29.4, 23.1],   // 채점자 3명 가중 총점 (0~30)
"essay_scoreT_avg": 25.666666              // 3명 평균 (현재 M5 학습 타겟)
```

### 슬롯 구조 명세

| 키 | 슬롯 개수 (고정) | 채점자 수 | 점수 범위 |
|---|---:|---:|---|
| `essay_scoreT_exp` | 3 | 3 | 0~3 정수 |
| `essay_scoreT_org` | 4 | 3 | 0~3 정수 |
| `essay_scoreT_cont` | 4 | 3 | 0~3 정수 |

**학년군별로 변하는 것은 슬롯 개수가 아니라 각 슬롯의 가중치 (0~여러 값)**. 가중치 0인 슬롯의 점수는 정규화에서 기여 없음. JSON 슬롯 자체는 항상 3/4/4로 고정.

### `rubric` 필드 (모든 JSON에 존재)

모든 22편 샘플에서 `rubric` 필드 확인. 구조 (데이터 설명서 PDF 6페이지 6.1~6.6.5):

```json
"rubric": {
  "essay_grade": "고등_2학년",       // student.student_grade와 동일
  "essay_type": "찬성반대",
  "essay_main_subject": "익명성에 대한 본인의 의견",
  "expression_weight": {
    "exp": 3,                       // 대분류 가중치
    "exp_grammar": 2,
    "exp_vocab": 4,
    "exp_style": 4
  },
  "organization_weight": {
    "org": 3,
    "org_paragraph": 3,             // org_paragraph=3이면 고2~고3, =0이면 중2~고1
    "org_essay": 3,
    "org_coherence": 3,
    "org_quantity": 1
  },
  "content_weight": {
    "con": 4,
    "con_clearance": 3,
    "con_description": 3,
    "con_novelty": 3,
    "con_prompt": 1                 // 수필형은 0
  }
}
```

→ multi-task 학습 시 **각 fold/에세이별로 가중치를 동적으로 로드**할 수 있도록 본 필드 활용 권장.

### ⚠️ 명시되지 않은 매핑 (검증 시 발견)

`essay_scoreT_exp[0..2]`, `org[0..3]`, `cont[0..3]`의 **각 인덱스가 어느 sub-rubric에 해당하는지 PDF/데이터 설명서 어디에도 명시되지 않음**.

가장 유력한 가설 (PDF 표 행 순서 기반):
- `exp`: [grammar, vocab, style]
- `org`: [paragraph, essay, coherence, quantity]
- `cont`: [clearance, description, novelty, prompt]

다양한 학년·타입 데이터로 매핑을 역추정한 별도 검증 노트북이 필요합니다.

## 5. 문단(paragraph) 채점 (별도)

### PDF 5페이지 spec

> "문단 루브릭의 경우 에세이 루브릭과 달리 '구성'과 '내용'을 평가할 수 없기 때문에 에세이 루브릭의 **'표현' 평가 항목만으로 채점**을 진행한다."

### JSON 구조 (22편 전체 일치)

```json
"paragraph_score": [{
  "paragraph_id": "001",
  "paragraph_scoreT": [2.8, 2.8, 2.4],
  "paragraph_scoreT_detail": {
    "paragraph_scoreT_exp": [[2,3,3], [2,3,3], [2,2,3]]
    // 다른 키 없음
  },
  "paragraph_scoreT_avg": 2.6666667
}]
```

단일 문단 에세이(ESSAY_69855)는 `paragraph_scoreT_exp == essay_scoreT_exp`. 다문단 에세이는 두 값이 분리되므로 **paragraph 채점을 auxiliary task로 활용 가치** 있음.

## 6. 현재 모델과의 차이

| 영역 | 데이터셋 spec | 현재 M5 (`pipelines/train.py` 검증) |
|---|---|---|
| 학습 타겟 | 9~11개 소분류 × 3 채점자 (≈ 99개 정보) | `essay_scoreT_avg` 단일 스칼라 |
| 출력 차원 | 3 대분류 또는 9~11 소분류 가능 | `num_labels=1` (`train_transformer.py:79~83`) |
| 가중치 처리 | 타입/학년군별 동적 (JSON `rubric` 필드에 캡슐화) | 무시 (이미 합쳐진 평균만 학습) |
| 평가 단위 | 소분류 / 대분류 / 종합 모두 가능 | 종합만 |
| 채점자 분산 | 3명 점수 보존 → uncertainty 정보 | 평균으로 압축, 분산 손실 |
| 문단 채점 | `paragraph_score`로 별도 | 미사용 |

## 7. 루브릭별 multi-task 채점 도입 시 활용 가능한 정보

| 정보 | 활용 |
|---|---|
| `essay_scoreT_org/cont/exp` (대분류 채점자별) | **대분류 3차원 multi-task 학습 타겟** |
| 각 채점자별 소분류 배열 | **소분류 9~11차원 multi-task 학습 타겟** |
| `rubric` 필드 (모든 JSON에 존재) | 타입/학년군별 가중치 동적 로드 |
| `paragraph_score` | 문단 단위 표현 채점 (auxiliary task) |
| 채점자 3명 분산 | 라벨 신뢰도 weighting (uncertainty-aware loss) |
| `essay_grade` (rubric 안) | 학년군 매핑 자동화 (PDF spec 의존 불필요) |

### 권장 학습 방식

```python
# 의사 코드: JSON rubric 필드에서 가중치 동적 로드
for essay in batch:
    rubric = essay["rubric"]
    weights = (
        list(rubric["expression_weight"].values())[1:] +    # 0은 대분류 가중치
        list(rubric["organization_weight"].values())[1:] +
        list(rubric["content_weight"].values())[1:]
    )
    # multi-task loss: weighted MSE over all active sub-rubrics
    loss = sum(w * mse(pred_i, true_i) for w, pred_i, true_i in zip(weights, pred, true) if w > 0)
```

이 방식은 PDF spec drift (초5 논술형 등) + 미명시 학년 (중3, 고3)을 모두 자동 처리.

## 8. 결론

**데이터셋은 루브릭 기반 다차원 채점이 핵심**입니다. 현재 M5는 이 풍부한 정보를 평균 1개로 축소해서 학습하는 상태로, 실제로는 데이터셋의 의도된 채점 단위를 충분히 활용하지 못하고 있습니다.

루브릭별 점수가 목표라면 즉시 도입 가능:
- 모든 항목별 점수가 JSON에 보존됨 → multi-task 학습 즉시 가능
- 가중치는 JSON `rubric` 필드에서 직접 로드 (PDF spec drift 회피)
- `TARGET_NAME` 단일 → 다중 + M5 head `num_labels=3` (대분류) 또는 `num_labels=9~11` (소분류)
- `evaluate.py`에 per-rubric metric + per-rubric fairness gate 추가
- MILESTONE/AGENTS acceptance criteria 차원별 재정의

## 10. 추가 검증 발견 (data quality 신호)

검증 에이전트가 22편 샘플링 중 발견한 추가 사실:

1. **중3 학년이 데이터셋에 존재** (PDF spec에는 명시 없음) — 자동으로 중2~고1 가중치 사용
2. **고3 학년은 수필형/대안제시에만 존재** — 논술형 고3은 0편
3. **초5 논술형 데이터의 `rubric` 가중치가 PDF spec과 다름** (data spec drift)
4. **JSON 슬롯 → sub-rubric 매핑이 PDF/데이터 설명서 어디에도 명시되지 않음** (역추정 필요)

→ 권장: PDF spec 기반이 아닌 **JSON `rubric` 필드를 진실의 원천(source of truth)으로 사용**.
