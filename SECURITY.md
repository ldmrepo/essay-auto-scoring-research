# Security Policy

## Supported Versions

Security review applies to the current `main` branch.

## Reporting a Vulnerability

Do not open a public issue for secrets, credentials, data exposure, or model
artifact leakage. Report the issue privately to the repository owner.

Include:

- affected commit or branch
- reproduction steps
- impacted file paths or artifacts
- whether secrets, private data, or external compute credentials may be exposed

## Data Handling

This repository may contain sample research data and generated artifacts. Code
is MIT licensed, but datasets and source documents remain subject to their
original provider terms. Verify redistribution rights before reusing data files.
